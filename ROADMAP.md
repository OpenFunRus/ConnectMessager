# ROADMAP - Connect Messager

Этот файл фиксирует план развития Connect Messager по этапам.

## Progress Snapshot - 2026-04-01

- Закрыта еще одна крупная волна production-polish вокруг `server-view`:
  - реакции на сообщения с emoji picker и toggle-flow;
  - разрешение реакций всем пользователям;
  - смена собственной аватарки и протяжка реальных аватаров в чат/сайдбар;
  - звуки отправки и получения сообщений;
  - экран реконнекта с countdown, кнопкой `Обновить страницу` и случайным первым интервалом `1-60 секунд`.
- После этого Connect Messager уже имеет не только DM и общую группу, но и кастомные role-filtered группы, reactions-flow и более зрелый reconnect UX.

## Stage 0 - База (сделано)

- Перенесена кодовая база в корень проекта (`apps/*`, `packages/*`).
- Выполнен базовый ребрендинг под Connect Messager.
- Настроена сборка Windows `exe` (`connectmessager-server.exe`).
- Добавлен скрипт `build-exe.bat` (остановка процессов + сборка).
- Подготовлен и развивается UI-прототип в `test`.
- Добавлена отдельная страница входа `test/login.html`.

## Stage 1 - MVP (текущий фокус)

- Завершить и стабилизировать новую схему входа:
  - первый разработчик;
  - одноразовые коды доступа для роли;
  - одноразовые коды входа для существующего пользователя;
  - cleanup временных пользователей по TTL.
- Убрать остаточную путаницу между legacy `invites` и новой access-code моделью в UI/документации.
- Связать прототипные UI-паттерны из `test` с основным клиентом (`apps/client`).
- Обновить логотипы/иконки приложения в `apps/client/public`.
- Зафиксировать единый стиль компонентов (кнопки, поля, карточки, попапы).
- Интегрировать в `apps/client` зрелую emoji-систему (Twemoji, copy/paste, recent, стабильный ввод без расхождений каретки).
- Завершить UX-polish поведения чат-ленты (chunk window, кнопка вниз, непрочитанные, переходы).
- Зафиксировать и стабилизировать новый group-management flow:
  - создание/редактирование/удаление групп;
  - фильтры видимости;
  - права управления группами;
  - сборка свежего `exe` с миграцией `0014`.
- Зафиксировать production-ready UX для:
  - реакций на сообщения;
  - смены своей аватарки;
  - reconnect-экрана;
  - звуков сообщений.

## Stage 2 - Core Messaging

- Личные и групповые чаты.
- Отправка/редактирование/удаление сообщений.
- Закрепы, уведомления, непрочитанные, поиск по сообщениям.
- Вложения (файлы/медиа) и базовые ограничения.
- Статусы пользователей и индикаторы присутствия.
- Упоминания, typing, цитаты, реакции и role-aware group visibility уже частично реализованы и находятся в фазе стабилизации, а не только в планах.

## Stage 3 - Corporate Features

- Роли и права (куратор города, модератор, сотрудник).
- Админ-панель организации и филиалов.
- Логи действий и базовый аудит.
- Настройки политик хранения и лимитов.
- Подготовка онбординга для новых филиалов.

## Stage 4 - Realtime & Calls

- Подключение голосовых каналов.
- Базовые звонки/видеозвонки.
- Демонстрация экрана.
- Диагностика качества связи и fallback-механизмы.

## Stage 5 - Release & Operations

- CI/CD для сборок и релизов.
- Подпись и публикация Windows-сборок.
- Документация для деплоя (Docker/локально).
- Мониторинг, резервное копирование, recovery-процедуры.
- Подготовка бета-релиза и план перехода в production.

## Ближайшие задачи (short list)

- Проверить свежую Windows-сборку на чистой машине:
  - старт после полной очистки `%appdata%`;
  - применение миграции `0014_group_channels`;
  - создание/редактирование/удаление группы;
  - экран реконнекта с countdown и reload-кнопкой.
- Добавить smoke/e2e сценарии для новых production-функций:
  - delete user -> history/files cleanup;
  - mention -> bell -> jump -> clear notification;
  - quote -> send -> jump to source;
  - group filter change -> realtime hide/show for affected users;
  - typing in group with 2-3 simultaneous users;
  - reactions add/toggle for 2+ users;
  - own avatar change -> avatar visible in sidebar and chat;
  - reconnect screen -> first random retry, then 60-second retries.
- Зафиксировать единый контракт GIF-сообщений (`cmx-gif-url:`) и quote-токенов (`:::cmx-quote|...:::`), включая совместимость со старыми клиентами.
- Продумать следующий этап GIF: превью ошибок битого URL, ретраи/таймауты рендера, безопасная валидация доменов.
- Добить production-ready поведение access-code flow:
  - отзыв/ревокация уже выданных JWT-сессий при необходимости;
  - понятные тексты ошибок и handoff между устройствами;
  - smoke-тесты сценариев role-code/user-code.
- Доделать ребрендинг статических ассетов (иконки, название окна, строки в UI) и вынести версию проекта в единый источник.
- Проверить production-сценарий определения клиентского IP за reverse proxy (Nginx/Caddy/Cloudflare) и зафиксировать рабочий конфиг заголовков.
- Добавить e2e smoke-тест сценария: login -> server down -> auto-restore -> reconnect без ручного входа.
- Добавить smoke-проверку механизма авто-reload клиента по `buildId` после server restart/deploy.
